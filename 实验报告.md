# 图书推荐系统实验报告

## 1. 项目概述

这次实验做的是一个简单的图书推荐系统。数据集使用的是 `Book-Crossing`，程序可以读取用户信息、图书信息和评分信息，然后给出推荐结果。

为了保证后面答辩时自己能讲清楚，我没有选太复杂的模型，而是用了比较基础的思路：先做数据清洗，再做热门推荐和基于用户评分的简单协同过滤推荐。

## 2. 选题背景与目标

推荐系统的作用是从很多物品里挑出用户可能感兴趣的内容。图书推荐比较适合做课程实验，因为数据结构直观，结果也容易看懂。

这次实验的目标有三个：

1. 能正确读取并处理图书评分数据。
2. 能输出一个用户的推荐结果。
3. 项目结构清楚，运行简单，方便提交和答辩。

## 3. 数据集说明

本项目使用 `Book-Crossing` 数据集，主要包含三张表：

- `BX-Users.csv`：用户信息
- `BX-Books.csv`：图书信息
- `BX-Book-Ratings.csv`：用户评分信息

原始数据中一共有：

- 用户约 `278858` 个
- 图书约 `271360` 本
- 显式评分 `433671` 条

这里的显式评分指评分值大于 0 的记录，因为 0 更像是浏览或隐式行为，不适合直接拿来做基础评分推荐。

## 4. 推荐方法说明

本实验用了两种很基础的方法。

### 4.1 热门图书推荐

先统计每本书被评分的次数和平均分，把评分次数较多、平均分较高的图书排在前面。这个方法不需要知道具体用户是谁，适合做冷启动或者演示兜底。

### 4.2 基于用户评分的简单协同过滤

核心想法是：如果两个用户给很多相同的书打过相近的分，那他们的阅读兴趣可能比较接近。

实现时的基本流程是：

1. 取出目标用户已经评分过的图书。
2. 找出也给这些图书打过分的其他用户。
3. 根据共同评分记录计算相似度。
4. 选出最相近的一批用户。
5. 从这些相似用户喜欢的书里，挑出目标用户还没看过的图书做推荐。

这个方法的优点是比较容易解释，缺点是受数据稀疏影响比较大，所以我又保留了热门推荐作为兜底方案。

## 5. 实现步骤

整个项目的实现步骤比较直接：

1. 读取三张 CSV 数据表。
2. 只保留评分大于 0 的记录。
3. 去掉重复评分。
4. 清理部分异常年龄和图书标题里的特殊字符。
5. 过滤掉评分次数太少的用户和图书。
6. 生成热门图书榜单。
7. 对指定用户做协同过滤推荐。
8. 把推荐结果打印到终端，并保存到文本文件。

## 6. 运行方法

先安装依赖：

```bash
python3 -m venv .venv
.venv/bin/pip install -r requirements.txt
```

运行主程序：

```bash
.venv/bin/python main.py
```

如果想指定用户：

```bash
.venv/bin/python main.py --user-id 11676
```

程序运行后会在 `output/demo_result.txt` 里保存一份结果，答辩时可以直接展示。

## 7. 实验结果展示

本项目默认做了较简单的过滤：

- 用户至少有 `10` 条有效评分
- 图书至少有 `20` 条有效评分

清洗后实际参与推荐的数据大约是：

- 用户 `6358` 个
- 图书 `2178` 本
- 评分 `60133` 条

默认演示时，程序会自动选择一个评分记录较多、能稳定生成推荐结果的用户。以实际运行结果为例，系统能够输出：

- 个性化图书推荐列表
- 热门图书推荐列表

推荐结果中可以看到一些较常见的高分书，比如《Harry Potter》《To Kill a Mockingbird》《Life of Pi》等，说明程序已经能根据评分信息给出比较合理的候选结果。

## 8. 学习路线 / 完成过程

这次实验我大概按下面的路线完成：

1. 先回顾推荐系统的基本概念，区分热门推荐和协同过滤。
2. 再看数据集结构，确认三张表分别表示什么。
3. 先做一个最简单能跑通的热门推荐。
4. 在热门推荐基础上补上用户协同过滤。
5. 最后再整理代码结构、运行方式和实验文档。

我觉得这样做的好处是不会一开始就把问题搞得很复杂，先保证程序跑通，再慢慢补完整。

## 9. 答辩时可能会被问到的问题与简短回答

### 问题 1：为什么选图书推荐？

因为图书数据比较清楚，用户、图书、评分三张表就能组成一个完整的推荐实验，结果也比较直观。

### 问题 2：为什么没有用深度学习模型？

这次实验更重视推荐流程的理解和实现，我选了更基础的协同过滤方法，代码简单，可解释性更强，更适合课程实验答辩。

### 问题 3：为什么要去掉评分为 0 的记录？

因为 0 在这个数据集中更像隐式行为，不是真正的用户打分。为了让评分推荐更清楚，这里只保留了大于 0 的显式评分。

### 问题 4：为什么要过滤掉评分次数太少的用户和图书？

因为评分太少时很难算出稳定的相似度，噪声会比较大。做一点简单过滤后，推荐结果会更稳定。

### 问题 5：你的推荐方法核心是什么？

核心就是找和目标用户评分习惯接近的人，再把这些相似用户喜欢、但目标用户还没看过的书推荐出来。

### 问题 6：如果某个用户几乎没有历史评分怎么办？

这种情况下个性化推荐不稳定，所以程序会退回到热门图书推荐。

## 10. 总结

这次实验完成了一个比较基础的图书推荐系统，包含数据读取、数据清洗、热门推荐、用户协同过滤推荐和结果输出几个部分。

虽然方法不复杂，但优点是结构完整、容易理解、运行稳定，也比较适合作为课程实验展示。
