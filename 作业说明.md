# 基于用户邻域的 Top-N 推荐作业说明

本次作业使用 `MovieLens 100K` 数据集，实现了一个典型的基于用户邻域的协同过滤 Top-N 推荐流程。为了便于交作业和后续整理，本实验的代码、说明、报告和数据集都集中放在 `实验二_推荐系统_用户邻域TopN/` 目录中。

## 一、实验文件说明

- `实验代码.py`
  主程序，包含数据集自动下载、数据读取、训练集测试集划分、用户相似度计算、Top-N 推荐生成和评价指标计算。
- `实验报告.md`
  实验报告文字版，可用于整理成最终提交材料。
- `作业说明.md`
  当前说明文件，用于说明实验结构、运行方式和数据集各文件用途。
- `数据集/`
  数据集目录。程序第一次运行时会自动下载并解压 `MovieLens 100K` 数据集。

## 二、本实验实际使用到的数据文件

虽然 `MovieLens 100K` 解压后会包含很多文件，但本实验代码直接读取的核心文件只有以下两个：

- `u.data`
  完整评分数据文件。每一行表示一条评分记录，格式为：
  `user id    item id    rating    timestamp`
  本实验从该文件中读取用户对电影的评分，并将评分大于等于 4 的记录视为正反馈。
- `u.item`
  电影信息文件。每一行包含电影编号、电影名称、上映日期、IMDb 链接以及电影类型标记等信息。
  本实验用它来把推荐结果中的电影编号转换成电影名称，便于展示推荐列表。

也就是说，如果老师只要求提交“数据集 + 实验代码”，通常提交：

- `实验代码.py`
- `数据集/u.data`
- `数据集/u.item`

就足够完成这次实验说明。

## 三、数据集里其他文件的作用

`MovieLens 100K` 是一个标准公开数据集，除了 `u.data` 和 `u.item` 之外，还自带了一些辅助文件。它们多数是为了做交叉验证、统计查看或补充用户属性而提供的，并不是本实验必须读取的核心文件。

- `README`
  数据集官方说明文档，介绍数据来源、授权、引用方式，以及每个文件的含义。
- `u.info`
  数据集规模统计信息，记录用户数、物品数和评分数。
- `u.genre`
  电影类别清单，例如 Action、Comedy、Drama 等。
- `u.user`
  用户属性信息，包含用户编号、年龄、性别、职业和邮编。
  如果做用户画像分析、冷启动研究，可以用到这个文件；本实验没有使用。
- `u.occupation`
  用户职业列表，是 `u.user` 中职业字段的补充说明。
- `u1.base` 和 `u1.test`
  第一组训练集和测试集。
- `u2.base` 和 `u2.test`
  第二组训练集和测试集。
- `u3.base` 和 `u3.test`
  第三组训练集和测试集。
- `u4.base` 和 `u4.test`
  第四组训练集和测试集。
- `u5.base` 和 `u5.test`
  第五组训练集和测试集。
  以上五组文件是官方给出的 5 折交叉验证划分，适合做更标准的实验评测。本实验没有直接使用它们，而是自行对 `u.data` 采用留一法划分训练集和测试集。
- `ua.base` 和 `ua.test`
  一组官方训练集和测试集划分，其中每个用户恰好有 10 条评分被放到测试集。
- `ub.base` 和 `ub.test`
  另一组与 `ua` 不重叠的训练集和测试集划分，也适合做实验对比。
- `allbut.pl`
  官方提供的脚本，用于生成“每个用户保留部分评分作测试集”的训练测试划分文件。
- `mku.sh`
  官方提供的 shell 脚本，用于从 `u.data` 重新生成这些训练集、测试集划分文件。
- `ml-100k.zip`
  原始压缩包文件，只是下载得到的压缩版本，不是实验中直接读取的数据文件。

## 四、为什么数据集看起来很大

很多同学第一次打开数据文件时会觉得内容很多，其实这是正常现象：

- `u.data` 一共有 `100000` 条评分记录，因此行数很多。
- `u.item` 虽然只有 `1682` 行，但每一行都包含电影名称、日期、链接和多个类型标记，所以看起来会比较长。

这并不表示数据集有问题，而是标准推荐系统数据集本身的正常形式。

## 五、实验流程

1. 读取 `u.data` 和 `u.item` 文件。
2. 将评分大于等于 4 的记录视为用户正反馈。
3. 对每个用户采用留一法划分数据集：
   将该用户最后一次正反馈放入测试集，其余正反馈放入训练集。
4. 根据用户共同喜欢的电影构建倒排表。
5. 计算用户两两之间的相似度矩阵。
6. 对目标用户选取最相似的 `K` 个邻居用户。
7. 汇总邻居喜欢但目标用户未看过的电影，计算候选电影得分。
8. 按得分排序，生成 Top-N 推荐结果。
9. 使用 `Precision`、`Recall`、`Coverage`、`Popularity` 指标进行评估。

## 六、运行方式

在本实验目录下执行：

```bash
python3 实验代码.py
```

如果想调整参数，例如邻居数和推荐数量，可以执行：

```bash
python3 实验代码.py --topk 30 --topn 10 --sample-user 10
```

## 七、可写入实验报告的简短结论

基于用户邻域的 Top-N 推荐方法首先通过用户历史行为构建兴趣集合，再通过共同兴趣计算用户之间的相似度。对于目标用户，系统从最相似的邻居中选出其喜欢但目标用户尚未接触的物品，并按加权得分排序后生成推荐结果。该方法实现简单、可解释性强，适合作为协同过滤推荐系统课程实验的基础方案。
