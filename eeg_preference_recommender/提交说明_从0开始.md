# EEG 脑信号推荐实验提交说明

这份说明是给第一次下载和提交项目的人看的，按下面步骤做就可以。

## 一、先下载项目

### 方法 1：会用 Git 的同学

打开终端，输入：

```bash
git clone -b codex/eeg-preference-recommender https://github.com/Szpiss/recommend-system.git
cd recommend-system/eeg_preference_recommender
```

这一步的意思是：从 GitHub 下载指定分支里的完整实验项目。

### 方法 2：不会用 Git 的同学

打开这个链接：

[https://github.com/Szpiss/recommend-system/tree/codex/eeg-preference-recommender](https://github.com/Szpiss/recommend-system/tree/codex/eeg-preference-recommender)

然后按下面做：

1. 点击绿色按钮 `Code`；
2. 点击 `Download ZIP`；
3. 下载完成后解压；
4. 打开解压后的文件夹；
5. 找到里面的 `eeg_preference_recommender` 文件夹。

后面提交和运行都围绕这个文件夹。

## 二、这个实验文件夹里有什么

需要重点看的文件夹是：

```text
eeg_preference_recommender/
```

里面已经包含完整实验内容：

```text
main.py                         一键运行入口
README.md                       项目说明
requirements.txt                依赖列表
src/                            实验源码
data/processed/                 已生成的模拟 EEG 数据
outputs/results/                训练和评估结果
outputs/figures/                可视化图片
outputs/checkpoints/            训练好的模型文件
report/experiment_report.md     实验报告
report/report_content.md        适合复制到 Word 的报告内容
```

## 三、如果老师只要求提交实验材料

最简单的提交方式：

把整个文件夹压缩成 ZIP：

```text
eeg_preference_recommender/
```

然后提交这个 ZIP 文件即可。

这个文件夹里已经包含：

1. 完整代码；
2. 模拟 EEG 数据；
3. 训练好的模型；
4. 评估指标；
5. 推荐结果；
6. 可视化图片；
7. 实验报告；
8. 运行说明。

## 五、如果老师要求现场运行

先进入项目目录：

```bash
cd recommend-system/eeg_preference_recommender
```

如果是下载 ZIP 的方式，就进入解压后的 `eeg_preference_recommender` 文件夹。

### Mac 或 Linux

执行：

```bash
python3 -m venv .venv
.venv/bin/pip install -r requirements.txt
.venv/bin/python main.py
```

### Windows

执行：

```bash
python -m venv .venv
.venv\Scripts\pip install -r requirements.txt
.venv\Scripts\python main.py
```

运行成功后，会看到这些结果文件：


这些就是实验结果、推荐结果、图表和报告。

## 七、报告怎么交

如果老师要 Markdown 报告，提交：

```text
report/experiment_report.md
```

如果老师要 Word，可以打开：

```text
report/report_content.md
```

把里面的内容复制到 Word 里，再调整封面、姓名、学号、课程名等信息。

## 八、最省心的提交方式

如果不确定老师具体要哪些文件，就直接提交：

```text
eeg_preference_recommender.zip
```

也就是把整个 `eeg_preference_recommender` 文件夹压缩后提交。

这样代码、数据、结果、图片、模型和报告都不会漏。

## 九、一句话总结

下载 GitHub 上的 `codex/eeg-preference-recommender` 分支，然后把里面的 `eeg_preference_recommender` 文件夹整个压缩提交即可。
